## 各文件功能说明

### 构建配置

| 文件 | 主要功能 |
|------|----------|
| `lightkv/CMakeLists.txt` | 顶层 CMake 构建配置，定义 C++20 标准、Debug(ASan)/Release 编译选项、源文件列表、库链接 (liblightkv.a)、测试开关 |

---

### 公共头文件 (`include/lightkv/`)

| 文件 | 主要功能 |
|------|----------|
| **`db.h`** | 数据库抽象接口：定义 `DB` 纯虚类，提供 `Put()`、`Delete()`、`Get()` 接口，以及静态工厂方法 `DB::Open()` |
| **`options.h`** | 配置参数结构体：`Options`（MemTable 大小、WAL 大小、Block Cache、块大小、压缩级别、Bloom 位数等）、`ReadOptions`（快照序列号、校验开关）、`WriteOptions`（同步写入开关） |
| **`status.h`** | 操作状态码：`Status` 类封装错误码（OK/NotFound/Corruption/I/O Error 等）及错误消息 |
| **`slice.h`** | 轻量级字节切片：`Slice` 类提供零拷贝字符串视图，支持比较、Hash（FNV-1a）、前缀移除、加法拼接 |
| **`encoding.h`** | 二进制编解码：Varint32/64 编解码、Fixed32/64 编解码、CRC32C 校验、长度前缀 Slice 编码 |
| **`skiplist.h`** | 无锁跳表：模板类 `SkipList` 实现内存有序索引（高度 12、分支因子 4），附带 `Arena` 内存池（4KB 块），支持插入/查找/删除标记 |
| **`memtable.h`** | 内存表：封装 `SkipList` 为 KV 数据库专用 MemTable，提供带序列号的插入/删除/查找 |
| **`block.h`** | 数据块：`Block` 类封装前缀压缩数据块的读取，`Block::Iterator` 提供 Seek/Next/Prev/二分查找，利用重启点数组加速定位 |
| **`block_builder.h`** | 数据块构建器：`BlockBuilder` 对 KV 条目进行前缀压缩编码，每 N 条记录写入重启点，输出紧凑的 Block 字节 |
| **`block_handle.h`** | 块位置句柄：`BlockHandle{offset, size}` 记录数据块在 SSTable 文件中的位置，支持 Encode/Decode |
| **`table_builder.h`** | SSTable 构建器：`TableBuilder` 负责写入数据块、构建索引块（每个数据块的最后 Key → BlockHandle）、构建 Bloom Filter、写入 Footer。附带 `FileWriter` 文件写入辅助类 |
| **`sstable.h`** | SSTable 读取器：`SSTable` 类打开 SSTable 文件（mmap），解析 Footer → Index Block → Bloom Filter，支持 `Get()` 按键查找（先 Bloom 过滤，再 Index 二分，最后 Data Block 线性扫描）及完整迭代器 |
| **`bloom_filter.h`** | 布隆过滤器：`BloomFilter` 实现双重哈希布隆过滤器，支持 `Add()` 添加键、`MayMatch()` 快速判存，bits_per_key 可配置 |
| **`cache.h`** | LRU 缓存：模板类 `LRUCache` 实现通用 LRU 缓存（mutex + list + unordered_map），`BlockCache` 封装为按 (file_id, block_offset) 索引的数据块缓存 |
| **`compaction.h`** | 层压缩：`Compaction` 描述一次压缩任务（L0→L1...），`CompactionWorker` 执行合并压缩（当前为简化桩实现） |
| **`wal.h`** | 写前日志：`WALWriter` 使用 mmap 追加写入 WAL 记录（支持按需扩容），`WALReader` 使用 mmap 顺序读取恢复记录 |
| **`db_impl.h`** | 核心引擎实现：`DBImpl` 继承 `DB`，管理 MemTable(活跃) + Immutable MemTable + WAL + 7 层 SSTable 文件 + Block Cache + 后台线程（Flush/Compaction） |

---

### 源代码文件 (`src/`)

| 文件 | 主要功能 |
|------|----------|
| **`db_impl.cpp`** | 核心引擎实现：`DB::Open()` 工厂方法、`Initialize()` 创建目录/WAL恢复、`Put()` 写入 WAL → MemTable（触发 Flush）、`Get()` 先查 MemTable → Immutable MemTable → 逐层 SSTable、`FlushMemTable()` 将 Immutable MemTable 持久化为 Level-0 SSTable、`MaybeScheduleCompaction()` 级别触发策略、`BackgroundWork()` 后台线程循环、`SearchSSTable()` 在指定层文件中查找 |
| **`memtable.cpp`** | MemTable 实现：`Insert()` 向 SkipList 插入 KV、`InsertDeletion()` 写入删除标记、`Get()` 查询时检查序列号和删除标记 |
| **`block.cpp`** | Block 迭代器实现：`Seek()` 通过重启点二分定位 + 线性扫描精确定位、`ParseEntry()` 解析前缀压缩条目（shared/non_shared/value_len）、`Next()/Prev()` 遍历、`key()/value()` 键值提取 |
| **`block_builder.cpp`** | BlockBuilder 实现：`Add()` 计算与前一条目的公共前缀，写入压缩编码（shared + non_shared + value_len + key_suffix + value）、`Finish()` 写入重启点数组，标记完成 |
| **`block_handle.cpp`** | BlockHandle 编码：写入 2 个 Varint64（offset + size），解码时反向读取 |
| **`table_builder.cpp`** | TableBuilder 实现：`Add()` 写入数据块 + 更新 Bloom Filter，块满或键顺序跨越时 `FlushDataBlock()` 记录到 Index、`Finish()` 写入 Index Block → Bloom Filter → Footer（含 Magic Number） |
| **`sstable.cpp`** | SSTable 实现：`Open()` mmap 文件，读 Footer → 读 Bloom Filter → 读 Index Block → 收集 Data Block Handles、`Get()` 先 Bloom 过滤 → Index Block Seek → 读对应 Data Block → 内查找、`ReadBlock()` 读数据块（优先从 Cache 获取）、迭代器跨块扫描 |
| **`bloom_filter.cpp`** | BloomFilter：`Add()` 双重哈希设置位、`MayMatch()` 双重哈希检查位、构造时自动计算哈希函数数量 k_
| **`cache.cpp`** | BlockCache：`Insert()` 按 CacheKey 插入、`Lookup()` 查找返回、按 data.size 作为 charge 计费 |
| **`wal.cpp`** | WAL 实现（mmap）：`WALWriter::Append()` 写入记录（CRC + Length + Payload），文件满时 `GrowFile()` 扩容至 2 倍、`WALReader::ReadRecord()` 逐条读取并校验 CRC，遇 kCRC_EOF 停止 |
| **`compaction.cpp`** | Compaction 桩实现：`DoCompaction()` 当前为空实现占位 |
| **`slice.cpp`** | Slice 工具函数（Hash 等轻量辅助） |
| **`status.cpp`** | Status::ToString() 将错误码转换为可读字符串 |
| **`iterator.cpp`** | DB 层迭代器占位（当前仅包含头文件引用，待实现） |

## 架构概览

```
                    ┌─────────────────────────────────────┐
                    │              DB 接口                  │
                    │    Put() / Get() / Delete()          │
                    └─────────────────┬───────────────────┘
                                      │
                    ┌─────────────────▼───────────────────┐
                    │            DBImpl (核心引擎)          │
                    │  ┌─────────────────────────────────┐ │
                    │  │  Write Path:                    │ │
                    │  │  WAL → MemTable(SkipList)       │ │
                    │  │         ↓ (满时Flush)            │ │
                    │  │  Immutable MemTable             │ │
                    │  │         ↓ (后台线程)              │ │
                    │  │  TableBuilder → Level-0 SSTable  │ │
                    │  └─────────────────────────────────┘ │
                    │  ┌─────────────────────────────────┐ │
                    │  │  Read Path:                     │ │
                    │  │  MemTable → Imm → SSTable(L0..6)│ │
                    │  │           Bloom → Index → Block  │ │
                    │  └─────────────────────────────────┘ │
                    │  ┌─────────────────────────────────┐ │
                    │  │  Background: Flush / Compaction  │ │
                    │  └─────────────────────────────────┘ │
                    └─────────────────────────────────────┘

存储格式:
┌──────────────────────────────────────────────────────┐
│ SSTable File Layout                                   │
│ ┌──────────┬──────────┬─────┬───────────┬──────────┐ │
│ │ Data     │ Data     │ ... │ Index     │ Bloom    │ │
│ │ Block 0  │ Block 1  │     │ Block     │ Filter   │ │
│ └──────────┴──────────┴─────┴───────────┴──────────┘ │
│                                              ┌──────┐ │
│                                              │Footer│ │
│                                              └──────┘ │
└──────────────────────────────────────────────────────┘

Block 编码格式:
┌────────────────┬──────────────────┬────┬──────────┐
│ KV Entries     │ Restart Offsets  │ #  │ Num      │
│ (prefix compr) │ (uint32 array)   │    │ Restarts │
└────────────────┴──────────────────┴────┴──────────┘
